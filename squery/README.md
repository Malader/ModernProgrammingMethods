Представление данных в S-выражениях

Требования

1. Определите формат представления данных с древовидной структурой в виде S-
выражений. В качестве семантической основы рекомендуется рассмотреть форматы

XML и/или JSON. Допускается использование как классических S-выражений, так и
расширенных, используемых в языке Clojure.

2. Разработайте язык навигации и простых запросов для заданного формата. Должны
поддерживаться: относительный или абсолютный путь к узлу дерева, путь с
условиями на свойства промежуточных узлов, путь с переменной вложенностью. В
качестве семантической основы рекомендуется рассмотреть язык XPath для XML.
Реализуйте функции поиска и модификации заданного формата посредством этого
языка.

3. Разработайте представление схемы (по аналогии с XML Schema). Реализуйте функции
проверки документа по схеме.
Дополнительные требования
• Разработайте язык трансформации в HTML, как упрощенный аналог XSLT.
• Реализуйте модель разбора по аналогии с XML SAХ, обеспечьте поддержку
трансформации и валидации по схеме в этом режиме.
Рекомендации
Ознакомьтесь XML-технологиями (SAX, DOM, XML Schema, XPath, XSLT)

Вот дополнительные требования:

1) В качестве формата для документов я буду использовать классические и расширенные Clojure-S-выражения  
2) Для навигации будет реализована поддержка абсолютных и относительных путей, условий на свойства промежуточных узлов, переменной вложенности.
А также будет реализован поиск и модификации по этому языку
3) Описывать схему я буду по аналогии с XML Schema и далее буду реализовать проверку документа по получившейся схеме.

## Структура проекта

Проект оформлен как Leiningen-проект на Clojure.

```text
project.clj
src/
  squery/core.clj       ;; демо: примеры навигации, модификации, валидации и HTML-трансформации
  squery/model.clj      ;; формат документов в виде S-выражений и базовые операции с деревом
  squery/path.clj       ;; язык путей (абсолютные/относительные, условия, переменная вложенность)
  squery/query.clj      ;; функции select / update* для поиска и модификации по путям
  squery/schema.clj     ;; представление схемы и валидация документа по схеме
  squery/html.clj       ;; упрощённый язык трансформации в HTML (аналог XSLT)
test/
  squery/model_test.clj
  squery/query_test.clj
  squery/schema_test.clj
````

## Требования

* Java JDK (рекомендуется 17)
* Leiningen (`lein` должен быть доступен в PATH)

Проверка:

```powershell
java -version
lein -v
```

## Как запустить проект

1. Перейти в каталог проекта:

   ```powershell
   cd "\ModernProgrammingMethods\squery"
   ```

2. Подтянуть зависимости (делается один раз):

   ```powershell
   lein deps
   ```

3. Запустить тесты:

   ```powershell
   lein test
   ```

4. Запустить демо-пример:

   ```powershell
   lein run
   ```
